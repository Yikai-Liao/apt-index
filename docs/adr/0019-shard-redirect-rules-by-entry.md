# Shard redirect rules by entry

Redirect rules are generated as static shards under the APT component and software entry instead of one global exact-path table. The deployable tree publishes per-entry shards and does not keep a production `redirect_rules.json` global table or a redirect-rules manifest. Each shard is a direct JSON map from package filenames to upstream package file URLs; the component and software entry are carried by the shard path rather than repeated in every key. This does not change the virtual package download paths published in APT `Packages` metadata; it only changes how the Worker locates redirect data for those existing paths. The Worker does not keep a fallback lookup for older global redirect data, because package download paths are generated by this project and strict routing keeps per-request redirect lookup bounded to the relevant entry shard.
