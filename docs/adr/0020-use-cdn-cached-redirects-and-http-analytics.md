# Use CDN-cached redirects and HTTP analytics

Package download redirects should be cacheable `302` responses with a long shared TTL rather than per-request Worker accounting events. Redirect rules remain static lookup data; the project should not introduce a separate rules cache as the primary optimization. The deployable tree publishes `/redirect-rules/snapshot.json.zst`, a zstd-compressed UTF-8 JSON redirect snapshot for release-time comparison, while the Worker reads per-entry redirect shards and stores the generated `302` response in Cloudflare's Cache API. The project should not use Cloudflare Pages `_redirects` for package redirects, because that would turn every package artifact path into a static redirect rule and reintroduce Pages redirect-count limits. The snapshot has a versioned top-level object with a `redirects` map from full package download paths to upstream package file URLs. Redirect shards and the snapshot should be served as static assets with conservative browser caching and long Cloudflare edge TTLs, because the Worker is the dominant reader. The Worker should also cache package-miss `404` responses briefly to avoid repeatedly loading the same shard for scanners or repeated bad paths. The publish workflow should fetch the previously deployed snapshot from Cloudflare and actively purge cached package download URLs whose redirect target changed, newly added package URLs that may hold stale negative cache entries, affected redirect shard assets, and the redirect snapshot asset. Download statistics are derived from Cloudflare HTTP request analytics over `GET /pool/*` package download paths, because Worker runtime telemetry is not the public download-count source of truth. `HEAD` requests are excluded from public download counts because they can represent probes, checks, or cache validation rather than a package download. Cloudflare's Worker Cache API does not honor `stale-while-revalidate` or `stale-if-error` directives, so those directives should not be treated as part of the redirect `302` cache design.
